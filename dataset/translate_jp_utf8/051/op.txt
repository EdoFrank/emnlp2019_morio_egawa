op_title: 私は、Microsoft Visual Studio 1998 が現在のオープンソース開発環境よりずっと先進的だったと思います。

私は6年間くらい　C++ の開発をしてから、 Visual Studio .NET の開発を６年間してきました。MS Visual Studio は私が使ってきた中でだんとつ一位の開発環境です。私は以前に JavaScript、 Node、 Ruby on Rails そして Clojure を使ってみたことがあったのですか、最近は、話題の「テキストエディタしか使わない」オープンソースの開発にハマっています。追記: ここでは Java の話をしていません。使ったことがないし、VSとおなじくらいけなされていると思います。

それらを使っていると、まるで20年前に戻っているみたいです。まあ、最低でも15年前。少なくともその頃にはデバッグ、編集そして再開の機能はありましたけど。Chrome の開発ツールは15年経ってようやく追いついてきているのです！ほとんどの人は未だに開発と編集を別々で行なっています、1990年代スタイルで！

今どきの新しいオープンソースの諸言語は全て一時的な熱狂で、ほとんどの人はインテリセンスが何なのかすら知らないんじゃないかと思います。

というのも冗談ですよ。私もこれらのオープンソースの言語は好きですよ。しかし、優れている点はどこにあるのでしょう？
